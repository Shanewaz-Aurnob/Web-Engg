Meeting Management API

Access and manage meeting details, agendas, attendees, and more through the provided endpoints.

Available Endpoints

1. POST /create-meeting

Create a new meeting with details, attendees, and agenda.

  • Response:
    • Success (200): Returns a success message upon successful creation.
    • Invalid Data Type (400): If the provided data is not valid.
    • Internal Server Error (500): If there's a server-side issue.

2. GET /agenda/:meeting_id

Retrieve the agenda for a specific meeting.

  • Response:
    • Success (200): Returns meeting details.
    • Invalid Data Type (400): If the meeting ID is not a number.
    • Not Found (404): If the meeting is not found.
    • Internal Server Error (500): If there's a server-side issue.

3. GET /attendees/:meeting_id

Retrieve the list of attendees for a specific meeting.

  • Response:
    • Success (200): Returns the list of attendees.
    • Invalid Data Type (400): If the meeting ID is not a number.
    • Not Found (404): If the meeting is not found.
    • Internal Server Error (500): If there's a server-side issue.

4. GET /meetingInfo

Retrieve the history of all meetings.

  • Response:
    • Success (200): Returns the details of all meetings.
    • Invalid Data Type (400): If the provided data is not valid.
    • Internal Server Error (500): If there's a server-side issue.

5. GET /meetingInfo/:meeting_id

Retrieve all details for a specific meeting.

  • Response:
    • Success (200): Returns all details for the meeting.
    • Invalid Data Type (400): If the meeting ID is not a number.
    • Not Found (404): If the meeting is not found.
    • Internal Server Error (500): If there's a server-side issue.

6. PUT /save-minutes

Save the minutes of a meeting to the database.

  • Response:
    • Success (200): Returns a success message upon successful update.
    • Invalid Data Type (400): If the provided data is not valid.
    • Not Found (404): If the meeting is not found.
    • Internal Server Error (500): If there's a server-side issue.

7. GET /get-minutes/:meeting_id

Retrieve the minutes URL for a specific meeting.

  • Response:
    • Success (200): Returns the minutes URL.
    • Invalid Data Type (400): If the meeting ID is not a number.
    • Not Found (404): If the meeting is not found.
    • Internal Server Error (500): If there's a server-side issue.

8. PUT /save-signature

Save the signature of a meeting to the database.

  • Response:
    • Success (200): Returns a success message upon successful update.
    • Invalid Data Type (400): If the provided data is not valid.
    • Not Found (404): If the meeting is not found.
    • Internal Server Error (500): If there's a server-side issue.

9. GET /get-signature/:meeting_id

Retrieve the signature URL for a specific meeting.

  • Response:
    • Success (200): Returns the signature URL.
    • Invalid Data Type (400): If the meeting ID is not a number.
    • Not Found (404): If the meeting is not found.
    • Internal Server Error (500): If there's a server-side issue.

10. DELETE /delete-meeting/:meeting_id

Delete a specific meeting.

  • Response:
    • Success (200): Returns a success message upon successful deletion.
    • Invalid Data Type (400): If the meeting ID is not a number.
    • Not Found (404): If the meeting is not found.
    • Internal Server Error (500): If there's a server-side issue.

11. PUT /save-decisions/:meeting_id

Save the decisions for a specific meeting's agendas.

  • Request Parameters:
    • meeting_id: The ID of the meeting for which decisions are being saved (integer).
  • Request Body:
    • decisions: An array of decisions corresponding to each agenda item (array of strings).
  • Response:
    • Success (200): Returns a success message upon successful update.
    • Invalid Meeting ID (400): If the provided meeting ID is not a valid number.
    • No Agendas Found (404): If no agendas are found for the specified meeting ID.
    • Decision Mismatch (400): If the number of decisions does not match the number of agendas.
    • Invalid Data Type (400): If the provided data does not match the expected schema.
    • Internal Server Error (500): If there's a server-side issue or if updating any agenda item fails.

Example Requests

POST /create-meeting

Endpoint to create a new meeting with detailed information.

Example Request:

fetch('/api/create-meeting', {
method: 'POST',
headers: {
  'Content-Type': 'application/json',
},
body: JSON.stringify({
  meeting_id: 1,
  meeting_type: "Project Update",
  selected_date: "2024-07-15T10:00:00.000Z",
  agenda: [
    {
      topic: "Introduction",
      description: "Opening remarks and introductions",
      decision: "No decision"
    },
    {
      topic: "Project Update",
      description: "Detailed project update",
      decision: "Approve next steps"
    }
  ],
  room_name: "Conference Room A",
  selected_attendees: [1, 2, 3],
  department_id: 5,
  signature_url: "https://example.com/signature.png"
})
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error('Error:', error));

Response

Success (200):

{
        "message": "Meeting created successfully"
      }

{
        "name": "Invalid data type",
        "message": "Error details here..."
      }

{
        "message": "Internal server error",

      }

PUT /save-signature

  • Request Body:

  • Response:

      • {"message": "Meeting minutes updated successfully", "updated": { ...updated data... }}
    • Invalid Meeting ID (400): If the provided meeting ID is not a valid number.
      • {"message": "Invalid meeting ID"}
    • Invalid Data (400): If the provided data does not match the expected schema.
      • {"name": "Invalid data type.", "message": "Error message details"}
    • Meeting Not Found (404): If no meeting is found with the specified ID.
      • {"message": "No meeting found with the specified ID"}
    • Internal Server Error (500): If there's a server-side issue.
      • {"message": "An unknown error occurred"}

PUT /save-decisions/:meeting_id

Updates decisions for agendas in a meeting.

Parameters

meeting_id
(integer) - The ID of the meeting.

Request Body

Requires a JSON object with the following structure:

                    
                {
                  "decisions": [
                    "decision1",
                    "decision2",
                    ...
                  ]
                }

Responses

  • 200 OK - Decisions updated successfully.
  • 400 Bad Request - Invalid meeting ID or number of decisions does not match number of agendas.
  • 404 Not Found - No agendas found for the specified meeting ID.
  • 500 Internal Server Error - An error occurred on the server side.
