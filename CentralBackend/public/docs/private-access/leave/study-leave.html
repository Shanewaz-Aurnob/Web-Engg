Study Leave Application Endpoint

Endpoints to manage study leave applications, including retrieval and submission.

Below are the available endpoints:

  • GET /: Retrieve paginated list of study leave applications.
  • POST /add: Submit a new study leave application.

Request Body Schema

Schema for the request body when adding a study leave application:


{
    "applicant_id": string,
    "applied_date": string (date in ISO format),
    "attachments": string | null,
    "department": string,
    "designation": string,
    "destination": string,
    "destination_country": string,
    "duration": number,
    "final_application": string | null,
    "financial_source": string,
    "joining_date": string (date in ISO format),
    "leave_start_date": string (date in ISO format),
    "my_application_chairman": string | null,
    "my_application_registrar": string | null,
    "name_of_program": string,
    "program_start_date": string (date in ISO format),
    "signature": string | null
}

Ensure all fields adhere to the specified data types and formats.
