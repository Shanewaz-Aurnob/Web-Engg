Exam Remuneration Endpoints

Manage different types of exam remuneration related activities including exam activities, bill calculation and much more.

Available Endpoints

1. GET /cuers

Retrieve all the roles and access privileges to the exam remuneration endpoints.

  • Response:
    • Success (200): Returns a list of roles & access privileges to exam remuneration endpoints for logged in user.
    • Internal Server Error (500): If there's a server-side issue.

Example Requests

GET /course-semester/

Request:

fetch('http://bike-csecu.com:5000/api/cuers',{
method: 'GET',
headers: {
    'Authorization': `Bearer ${sessionId}`
    'Content-Type': 'application/json',
    }                
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error('Error:', error));

Response:

{
"roles": {
    "staff_role": "chairman",
    "cec_role": "chairman-of-exam-committee",
    "evaluator_role": "evaluator"
    }
}

Each endpoint provides a specific functionality related to exam remuneration, provided the teachers or staff have their roles assigned. Look up each endpoint's documentation for detailed information on how to interact with the exam remuneration related features of the CSECU Web Server.
