Endpoint: /hall

Access hall information including details about specific halls within the system.

Available Endpoints

1. GET /hall/:id

Retrieve details of a specific hall by its ID.

  • Parameters:
    • id: Hall ID (integer)
  • Response:
    • Success (200): Returns hall details.
    • Not Found (404): If hall with the given ID does not exist.
    • Invalid Data Type (400): If the provided ID is not a number.
    • Internal Server Error (500): If there's a server-side issue.

2. GET /hall/

Retrieve a list of all halls.

  • Query Parameters:
    • page: Page number for pagination (optional)
    • limit: Number of results per page (optional)
    • sort: Sort halls by a specific field (optional)
  • Response:
    • Success (200): Returns a paginated list of halls.
    • Internal Server Error (500): If there's a server-side issue.

Example Requests

GET /hall/1

Request:

fetch('http://bike-csecu.com:5000/api/hall/1')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Response:

{
  "hall_id": 1,
  "name": "Hall A",
  "capacity": 150,
  "location": "Main Building, Floor 1",
  // Other hall fields
}

GET /hall/

Request:

fetch('http://bike-csecu.com:5000/api/hall/')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Response:

{
  "halls": [
    {
      "hall_id": 1,
      "name": "Hall A",
      "capacity": 150,
      "location": "Main Building, Floor 1",
      // Other hall fields

      "hall_id": 2,
      "name": "Hall B",
      "capacity": 200,

    // More halls...
  ],
  "pagination": {
    "totalPages": 3,
    "currentPage": 1,
    "nextPage": 2,
    "prevPage": null
  }
}

Additional Information

The /hall endpoint provides access to hall information such as name, capacity, and location. For pagination, use the page and limit query parameters to navigate through the results. Contact support for any issues related to hall endpoints.
