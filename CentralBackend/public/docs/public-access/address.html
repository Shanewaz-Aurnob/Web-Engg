Endpoint: /address

Access address information including details about specific addresses within the system.

Available Endpoints

1. GET /address/:id

Retrieve details of a specific address by its ID.

  • Parameters:
    • id: Address ID (integer)
  • Response:
    • Success (200): Returns address details.
    • Not Found (404): If address with the given ID does not exist.
    • Invalid Data Type (400): If the provided ID is not a number.
    • Internal Server Error (500): If there's a server-side issue.

2. GET /address/

Retrieve a list of all addresses.

  • Query Parameters:
    • page: Page number for pagination (optional)
    • limit: Number of results per page (optional)
    • sort: Sort addresses by a specific field (optional)
  • Response:
    • Success (200): Returns a paginated list of addresses.
    • Internal Server Error (500): If there's a server-side issue.

Example Requests

GET /address/1

Request:

fetch('http://bike-csecu.com:5000/api/address/1')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Response:

{
  "address_id": 1,
  "street": "123 Main St",
  "city": "Anytown",
  "state": "CA",
  "postal_code": "12345",
  // Other address fields
}

Additional Information

The /address endpoint provides access to address information such as street, city, state, and postal code. For pagination, use the page and limit query parameters to navigate through the results. Contact support for any issues related to address endpoints.
