Endpoint: /department

Access department information including details about specific departments within the system.

Available Endpoints

1. GET /department/:id

Retrieve details of a specific department by its ID.

  • Parameters:
    • id: Department ID (integer)
  • Response:
    • Success (200): Returns department details.
    • Not Found (404): If department with the given ID does not exist.
    • Invalid Data Type (400): If the provided ID is not a number.
    • Internal Server Error (500): If there's a server-side issue.

2. GET /department/

Retrieve a list of all departments.

  • Query Parameters:
    • page: Page number for pagination (optional)
    • limit: Number of results per page (optional)
    • sort: Sort departments by a specific field (optional)
  • Response:
    • Success (200): Returns a paginated list of departments.
    • Internal Server Error (500): If there's a server-side issue.

Example Requests

GET /department/1

Request:

fetch('http://bike-csecu.com:5000/api/department/1')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Response:

{
  "department_id": 1,
  "name": "Computer Science",
  "head_of_department": "John Doe",
  "university_id": 1
  // Other department fields
}

GET /department/

Request:

fetch('http://bike-csecu.com:5000/api/department/')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Response:

{
  "departments": [

      "name": "Mathematics",
      "head_of_department": "Jane Smith",
      "university_id": 1
      // Other department fields

    "totalPages": 5,

    "nextPage": 2,
    "prevPage": null
  }
}

Additional Information

The /department endpoint provides access to department information such as name, head of department, and associated university. For pagination, use the page and limit query parameters to navigate through the results. Contact support for any issues related to department endpoints.
