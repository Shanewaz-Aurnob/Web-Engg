Endpoint: /university

Access university information including details about specific universities within the system.

Available Endpoints

1. GET /university/:id

Retrieve details of a specific university by its ID.

  • Parameters:
    • id: University ID (integer)
  • Response:
    • Success (200): Returns university details.
    • Not Found (404): If university with the given ID does not exist.
    • Invalid Data Type (400): If the provided ID is not a number.
    • Internal Server Error (500): If there's a server-side issue.

2. GET /university/

Retrieve a list of all universities.

  • Query Parameters:
    • page: Page number for pagination (optional)
    • limit: Number of results per page (optional)
    • sort: Sort universities by a specific field (optional)
  • Response:
    • Success (200): Returns a paginated list of universities.
    • Internal Server Error (500): If there's a server-side issue.

Example Requests

GET /university/1

Request:

fetch('http://bike-csecu.com:5000/api/university/1')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Response:

{
  "university_id": 1,
  "name": "CSECU University",
  "location": "City A",
  // Other university fields
}

GET /university/

Request:

fetch('http://bike-csecu.com:5000/api/university/')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Response:

{
  "universities": [
    {
      "university_id": 1,
      "name": "CSECU University",
      "location": "City A",
      // Other university fields
    },
    {
      "university_id": 2,
      "name": "Another University",
      "location": "City B",
      // Other university fields
    },
    // More universities...
  ],
  "pagination": {
    "totalPages": 2,
    "currentPage": 1,
    "nextPage": 2,
    "prevPage": null
  }
}

Additional Information

The /university endpoint provides access to university information such as name and location. For pagination, use the page and limit query parameters to navigate through the results. Contact support for any issues related to university endpoints.
